项目组织：
源代码都是存放在GOPATH的src目录下，如果有多个项目的时候，是通过包来组织项目目录结构的。使用域名等不易冲突的包名来避免项目冲突。


go的执行原理：
源码编译之后的可执行文件安装时
1. 如果GOPATH只有一个工作区，那么可执行文件被存放在此工作区的bin文件夹内
2. 如果GOPATH有多个工作区，那么可执行文件被存放在GOBIN指向的目录下


所有的命令源码文件应该被单独放在一个代码包中。同一个包下不要存放多个命令源码文件(含main方法的文件)，这种方式只能分别使用go run ...执行，但是无法通过go build和go install
